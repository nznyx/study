# Потоки или процессы?

Это сущности OS, если нет OS то нет и таких абстраций

В рамках одного процесса можно выполнять несколько потоков

| Аспект | Процессы | Потоки |
|---|---|---|
| Начало выполнения | Процесс начинается с одного потока | В начале выполнения процесса существует всего один поток |
| Создание новых единиц выполнения | Процессы могут создавать дочерние процессы | Потоки внутри процесса могут создавать новые потоки |
| Адресное пространство | У каждого процесса своё адресное пространство; обращаться к памяти других процессов нельзя, иначе возможен Segfault | Все потоки одного процесса разделяют общее адресное пространство процесса |
| Права доступа к памяти | Изолированы от других процессов | Имеют доступ к общей памяти своего процесса |
| Родительские отношения | У процессов могут быть родители — другие процессы | Между потоками нет отношения «предок-потомок
| Преимущества | Безопасность, Налёжность | Скорость, Простота |

Ускорение ограничено [законом Амдала](laws/amdahls-law.md)
